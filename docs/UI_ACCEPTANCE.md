# UI 验收路径（小白点击流）

> 用于验证可视化界面各功能是否正常。双击 `start_ui.command` 后按以下步骤操作。

## 前置条件

- macOS 环境
- 已安装数据 fallback（`~/.agents/skills/a-share-data/` 目录存在）
- 首次运行会自动创建 `.venv` 并安装轻量依赖

## 验收步骤

### 1. 启动系统

双击项目根目录下的 **`start_ui.command`**，终端会显示：

```
========================================
  🏦 A 股智能选股系统
  仅供研究学习，不构成投资建议
========================================
```

自动检测 Python3 → 创建/确认虚拟环境 → 安装依赖 → 启动 Streamlit。

浏览器自动打开或手动访问 **http://localhost:8501**。

### 2. 首页引导

打开后首页应显示 **三步引导卡片**（而非空白页面）：

- ① 选参数
- ② 开始选股
- ③ 看结果

左侧栏参数面板自动展开，可见三个步骤编号。

### 3. 选股

1. 左侧栏「① 选择股票池」：保持默认 **static (55只精选)**
2. 扫描数量保持默认 20，展示数量保持默认 10
3. 数据起始日期保持默认 2024-01-01
4. 点击 **🚀 开始选股** 按钮

**预期**：
- 按钮点击后显示 "⏳ 正在选股中" 加载动画
- 完成后弹出绿色成功提示（如 "✅ 选股完成！成功 10/10 只"）
- 如果 akshare 不可用，提示 "数据来自备用源（akshare 暂不可用），覆盖可能不全"（非崩溃）

### 4. 查看候选表格

自动切换到 **🎯 候选表格** Tab：

- 显示 Top N 候选的排行表格
- 表格包含：排名、代码、名称、行业、评分、决策、风险、置信度、数据区间
- **数据区间** 列显示每只股票的实际数据起止日期
- **覆盖** 列显示 ✓（正常）或 ⚠️不全（覆盖不足）
- 决策列有颜色区分（绿色=强观察/浅绿=观察/红色=回避）
- 表格上方有 **数据覆盖摘要卡片**（数据范围、最少/最多/平均K线条数、覆盖不全数量）

### 5. 查看逐只详情

在候选表格下方，展开任意股票的详情：

- 显示评分、决策、风险、置信度
- 显示数据区间（actual_start ~ actual_end）
- 如果覆盖不全，显示 ⚠️ 警告及白话解释
- 显示六因子得分和因子数值

### 6. 查看验证摘要

切换到 **📋 验证摘要** Tab：

- 显示整体质量（🟢 good / 🟡 usable_with_caution / 🔴 poor）
- 整体质量下方有中文解释
- 显示总候选数、选股成功、覆盖不足率、平均分/最高分
- 显示置信度分布、决策分布、风险分布
- 显示行业分布柱状图

### 7. 运行历史复盘

1. 点击左侧栏 **📈 历史复盘** 按钮（需要先有选股结果，否则按钮灰色）
2. 等待加载（较慢，需要重新拉取数据）

**预期**：
- 加载完成后弹出成功提示
- 自动切换到 **📈 历史复盘** Tab
- 显示复盘汇总卡片（验证总数/跳过/Win/Flat/Loss + 平均收益/回撤/最佳/最差）
- 下方显示逐只复盘详情表格

### 8. 查看报告

切换到 **📄 报告** Tab：

- 自动生成完整 Markdown 日报
- 包含市场概况、验证摘要、批量选股结果、历史窗口复盘、免责声明
- 底部提示报告已保存路径

### 9. 确认数据覆盖提示

在整个流程中，应能直观看到：
- 候选表格的「数据区间」列
- 数据覆盖摘要卡片（数据范围、K线条数统计）
- 逐只详情中的覆盖不足警告
- 验证摘要中的覆盖不足率

**不需要翻日志就能了解数据覆盖情况。**
