## 三、结论

系统核心功能可用：小白能从首页三步引导开始，点击选股按钮，等待后看到候选表格。数据覆盖信息（区间/K线/警告）已在代码中完整实现。CLI 全链路验证通过。

**当前 MVP 可交付**，但需在 README 中明确告知：
- akshare 不稳定是已知限制，skill fallback 是主要数据源
- 选股等待 30-60 秒是正常现象
- coverage_warning 普遍出现是数据集较小导致的，不影响使用

## 四、P7.3 处理结果

P7.3 已把最容易让小白误判为“系统坏了”的问题转为明确提示：

- 首页显示当前数据通道状态，说明 `skill_fallback` 是可用的备用数据通道。
- 默认扫描数量从 20 只改为 10 只，降低首次等待时间。
- 选股按钮附近提示预计等待 30-60 秒。
- 覆盖不全提示改成白话说明：不是报错，系统会自动降低评分和置信度。
- 新增 `scripts/install_fallback.command`，给首次安装备用数据通道一个明确入口。

仍待后续继续优化：

- Tab 懒加载导致自动化验收不容易完整捕获验证摘要和报告页。
- `scripts/install_fallback.command` 仍依赖 GitHub 网络，离线环境需要额外方案。
