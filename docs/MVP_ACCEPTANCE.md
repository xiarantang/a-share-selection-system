## 二、仍然阻塞小白使用的问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| akshare 首次拉取经常失败 | 🔴 高 | 当前环境 akshare 100% 失败，全部依赖 skill_fallback（仅 ~120 条 K 线），小白可能以为系统坏了 |
| 选股等待时间较长 | 🟡 中 | 10 只股票 ~60 秒，新手可能以为卡死 |
| 数据覆盖普遍不足 | 🟡 中 | 所有股票都触发 coverage_warning，小白可能不理解这是正常现象 |
| Tab 内容需手动点击才渲染 | 🟡 中 | 验证摘要/报告的 Tab 懒加载，自动化验收需额外点击 |
| 首次无 fallback 时体验差 | 🟡 中 | 若未安装 Skill fallback，选股可能全部失败，提示不够醒目 |

## 三、结论

系统核心功能可用：小白能从首页三步引导开始，点击选股按钮，等待后看到候选表格。数据覆盖信息（区间/K线/警告）已在代码中完整实现。CLI 全链路验证通过。

**当前 MVP 可交付**，但需在 README 中明确告知：
- akshare 不稳定是已知限制，skill fallback 是主要数据源
- 选股等待 30-60 秒是正常现象
- coverage_warning 普遍出现是数据集较小导致的，不影响使用
