# P8 阶段路线图

> 规划日期：2026-05-16
> 前置版本：v0.5（小白可用 MVP 已发布）
> 总原则：在不破坏小白体验的前提下，渐进增强系统能力。

---

## 总览

| 阶段 | 主题 | 优先级 | 预估改动量 |
|------|------|--------|-----------|
| P8.1 | 数据质量增强 | 🟢 高 | 中 |
| P8.2 | 策略可解释增强 | 🟢 高 | 小 |
| P8.3 | UI 体验增强 | 🟡 中 | 中 |
| P8.4 | 策略管理 | 🟡 中 | 中 |
| P8.5 | AI 辅助解释 | 🔵 低 | 小 |

## P8.1 数据质量增强

### 目标
把选股数据从「约 120 条 K 线 / 大量覆盖不全」提升到「至少 250 条 K 线 / 覆盖率 > 80%」。

### 为什么做
当前 v0.5 最大痛点是数据覆盖不足——所有股票都标记「覆盖不全」，用户对结果信任度低。数据是选股系统的地基，地基不牢上层再好看也没用。

### 改动范围
- **数据源**：在现有 akshare + skill_fallback 基础上，增加新的免费数据源（如 baostock 补历史、easyquotation 补实时）
- **fallback 策略**：当前只降级到 skill_fallback（120 条），应增加多级 fallback 链
- **缓存策略**：增加缓存有效期、增量更新（不每次都全量拉取）
- **数据质量指标**：在 UI 中展示每条 K 线的数据来源和覆盖情况

### 验收标准
- [ ] select 跑 10 只 static 股票，至少 8 只有 250+ 条 K 线
- [ ] 「覆盖不全」比例从当前的 100% 降到 ≤ 20%
- [ ] 数据拉取失败时有明确的 fallback 来源提示
- [ ] CLI 全链路通过

### 禁止做什么
- ❌ 不删除现有 akshare / skill_fallback 数据源
- ❌ 不引入需要 API Key 的商业数据源
- ❌ 不改变选股评分逻辑

## P8.2 策略可解释增强

### 目标
让小白用户能看懂「为什么这只股票评分高、风险高、置信度低」。

### 为什么做
当前 UI 显示了评分、决策、风险等级，但缺少白话解释。小白看到 "strong_watch / medium_confidence / high_risk" 可能一头雾水。P7.8 的 TROUBLESHOOTING 已经做了术语表，但更好的方式是在结果页直接给出上下文相关的解释。

### 改动范围
- **app.py**：在逐只详情的 expander 中增加「大白话解释」段落（如 "评分 72 / 强观察：这只股票均线多头排列，趋势向好，但因为只有 120 条数据，置信度中等"）
- **strategies/selection.py**：在返回结果中增加 explain 字段（简短中文描述每个因子贡献）
- **不新增模块**

### 验收标准
- [ ] 每只股票的 expander 中有「📝 一句话解释」段落
- [ ] 解释中至少提到：最高贡献因子、拖后腿因子、置信度原因
- [ ] 解释用白话，不出现英文缩写（除非附中文翻译）
- [ ] CLI 全链路通过

### 禁止做什么
- ❌ 不修改评分公式 / 因子权重
- ❌ 不引入外部解释引擎

## P8.3 UI 体验增强

### 目标
让 Streamlit 界面更流畅、更直观，减少小白困惑。

### 为什么做
当前 UI 已经可用，但有一些体验瑕疵：选股中不知道进度、结果页 Tab 需要手动切换才能看到核心内容、首次使用检查区的状态有时候 show 有时候 hide 导致困惑。

### 改动范围
- **选股进度**：用 st.progress() 显示实时进度（当前只有 spinner）
- **结果页首屏**：把候选表格 Top 3 直接渲染在 Tab 外（当前首屏只有覆盖摘要，候选表格在 Tab 里）
- **状态一致**：统一就绪状态判定逻辑（当前 expander 里和外面的判定逻辑分开写的）
- **颜色和字体**：调整 risk_level 的颜色映射（高风险红色、中风险橙色、低风险绿色），让小白一目了然

### 验收标准
- [ ] 选股时有进度条（不是无限 spinner）
- [ ] 结果页首屏直接显示 Top 3 候选（不用切换 Tab）
- [ ] 就绪状态在 expander 展开和折叠时显示一致
- [ ] 高风险股票在表格中红色标记
- [ ] CLI 全链路通过

### 禁止做什么
- ❌ 不改变页面整体布局结构
- ❌ 不引入新的前端框架
- ❌ 不修改选股 / 验证逻辑

## P8.4 策略管理

### 目标
让系统支持多策略并存，用户可以在 UI 里切换策略。

### 为什么做
当前只有一个 6 因子评分策略。未来可能增加更多策略（如 MACD 金叉策略、趋势回踩策略等），需要策略管理框架。这为 P8.5 AI 辅助解释和未来的策略 A/B 测试打基础。

### 改动范围
- **strategies/registry.py**：完善策略注册机制（当前已有雏形）
- **app.py**：左侧栏增加策略选择下拉框
- **CLI**：`select --strategy <name>` 支持多策略
- **策略命名**：当前 6 因子策略命名为 "multi-factor-v1"
- **不新增策略逻辑**：本阶段只做框架，策略内容在后续阶段逐个添加

### 验收标准
- [ ] UI 左侧栏有策略选择器（默认 "multi-factor-v1"）
- [ ] CLI `select --strategy multi-factor-v1` 行为不变
- [ ] 切换策略不报错（即使只有 1 个策略）
- [ ] CLI 全链路通过

### 禁止做什么
- ❌ 不删除 / 重写现有选股逻辑
- ❌ 不新增实际策略（只做框架）
- ❌ 不改变现有 API / JSON 格式

## P8.5 AI 辅助解释

### 目标
用 AI 帮小白理解选股结果、解答「这只股票值得关注吗」类问题。

### 为什么做
P8.2 做了规则化解释（因子贡献），但规则化解释有条件限制——它只能解释「为什么评分高」，不能回答「这只股票在当前市场环境下值得关注吗」。AI 可以给出更灵活的上下文解释。

**但这是 P8 最后一阶段**，因为：
1. AI 依赖数据质量和策略稳定（P8.1 / P8.4 是前提）
2. AI 不能替代基础解释（P8.2 是前提）
3. AI 可能增加小白困惑（需要 UI 配合 P8.3）

### 改动范围
- **agent/bridge.py**：复用现有 agent 桥接，传入选股结果 + 市场上下文
- **app.py**：在逐只详情 expander 中增加「🤖 AI 怎么看」段落（可选展开，默认折叠）
- **不新增 AI 模型**：复用项目已有的 agent/ 模块，模型通过配置选择（默认关闭）
- **feature flag**：AI 解释默认关闭，在配置中开启，避免增加小白启动复杂度

### 验收标准
- [ ] 开启 AI 后，选股结果 expander 中有「🤖 AI 怎么看」段落
- [ ] AI 解释包含：风险提示、关注理由、注意事项
- [ ] AI 不可用时优雅降级（显示「AI 暂不可用，请查看规则解释」）
- [ ] AI 默认关闭，不增加小白启动步骤
- [ ] CLI 全链路通过

### 禁止做什么
- ❌ 不让 AI 参与评分或决策（只做解释）
- ❌ 不引入需要 API Key 的闭源模型作为唯一选项（支持本地模型）
- ❌ 不把 AI 解释作为默认行为
- ❌ 不在 AI 输出中包含投资建议
